## SSR (Server Side Rendering)

SSR은 React가 컴포넌트를 HTML 스트링으로 렌더링하고 (react-dom/server의 renderToString() 같은 API를 사용하여) 클라이언트로 보내는 것을 말한다.

클라이언트에서 React는 받은 HTML을 다시 렌더링하여 클릭 핸들러나 상호작용을 할 수 있게 추가하는 작업을 수행한다.

이 과정을 수화(hydration)이라고 한다.

즉, 상호작용 할 수 없는 화면을 빨리 보여주고나서 상호작용 할 수 있도록 다시 렌더링하는 방법이다.

브라우저는 클라이언트에서 다시 렌더링할 때 필요한 자바스크립트를 다운로드 받아야하므로 번들 사이즈가 줄어들지는 않는다.

## RSC (React Server Component)

RSC는 HTML 문자열로 렌더링하지 않고 React 엘리먼트로(객체, JSON같은 형태로) 클라이언트 측에 보낸다.

- 서버 컴포넌트의 코드는 서버에서만 실행되고 클라이언트에서 절대로 다운로드되지 않아서 번들 사이즈를 줄이고 시작 시간을 향상 시킬 수 있다.
- 서버 컴포넌트는 서버측 데이터 소스(database, file system)를 접근할 수 있다.
- 서버 컴포넌트는 클라이언트 컴포넌트와 원할하게 통합될 수 있다. 서버 컴포넌트가 서버에서 데이터를 로드하고 클라이언트 컴포넌트에 props로 보낼 수 있다는 의미
- 서버 컴포넌트는 어떤 클라이언트 컴포넌트를 렌더링할지 동적으로 선택할 수 있다.
- 서버 컴포넌트는 다시 로딩 했을 때 클라이언트 상태를 보존한다. 즉, 서버 컴포넌트 트리가 refetch 되어도 클라이언트 상태나 포커스 진행중인 애니메이션이 방해받거나 초기화되지 않는다.
- 서버 컴포넌트는 점진적으로 렌더링되며, Suspense와 결합하면 클라이언트에 UI 단위로 스트림 렌더링을 할 수 있다.
