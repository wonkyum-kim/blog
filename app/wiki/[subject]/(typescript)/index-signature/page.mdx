## 인덱스 시그니처

객체의 키를 정의하는 방식

<CodeSelector names={['ts']} desc='indexsignature' />

```ts
type Foo = {
  [key in 'hello' | 'hi']: string
}
```

`Record`를 사용해도 같은 결과가 나온다.

<CodeSelector names={['ts']} desc='record' />

```ts
type Foo = Record<'hello' | 'hi', string>
```

인덱스 시그니처를 사용했을 때, `Object.keys()` 사용을 주의한다.

<CodeSelector names={['ts']} desc='example' />

```ts
type Foo = {
  [key in 'hello' | 'hi']: string
}

const foo: Foo = {
  hello: 'hello',
  hi: 'hi',
}

// ❌ Error
Object.keys(foo).map((key) => {
  return hello[key]
})

// ✅ Good
Object.keys(foo).map((key) => {
  return hello[key as keyof Foo]
})
```

왜냐하면, `Object.keys()`의 반환 타입이 `string[]`인데, 타입스크립트에서 `string`은 객체의 키로 사용하지 못하기 때문이다.
