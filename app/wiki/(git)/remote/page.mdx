## git remote

로컬 저장소를 깃허브에 있는 저장소와 연결하거나 해제할 때 사용한다.

## origin

내 깃허브에 있는 저장소를 `origin`이라고 한다.

<CodeSelector names={['shell']} desc='add' />

```shell
git remote add origin https://github.com/....git
```

## upstream

내가 만약 다른 사람의 저장소를 fork하고 작업을 하는 경우가 있다.

다른 사람의 원래 저장소(원본)을 `upstream`이라고 한다.

즉 나는 upstream을 fork해서 origin에서 작업을 하게 된다.

## origin을 최신 상태로 유지하기

push 하기 전에 origin이 upstream의 최신 상태를 유지하도록 하는 것이 좋다.

<CodeSelector names={['shell']} desc='pull' />

```shell
git checkout dev
git pull upstream dev
```

이렇게 하면 로컬과 origin의 상태가 upstream과 동일해진다.
